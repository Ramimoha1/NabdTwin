# Sprint 1: KPI & Metric Calculation

## Overview

Sprint 1 focuses on the **Analytics Service** (`src/api/analytics/services/analytics.ts`) which calculates daily performance metrics at multiple organizational levels: Employee, Floor, and Branch. The service implements a **hierarchical aggregation pattern** where lower-level KPIs roll up into higher-level metrics.

---

## 1. Input Entities (Master Data)

These are the source entities queried by the Analytics service to calculate KPI values.

| Entity Name | Collection Name | Key Attributes | Type | Relationship |
|-------------|-----------------|----------------|------|--------------|
| **Branch** | `branches` | `name`, `code`, `address`, `city`, `country`, `revenueTarget`, `branchStatus` | Master | Has Many → Floors, Employees, Departments, Teams |
| **Floor** | `floors` | `name`, `floorNumber`, `totalArea`, `model3dUrl` | Master | Belongs To → Branch; Has Many → Workspaces, Employees |
| **Workspace** | `workspaces` | `name`, `workspaceType`, `capacity`, `area`, `positionX/Y/Z` | Master | Belongs To → Floor; Has Many → Employees |
| **Employee** | `employees` | `firstName`, `lastName`, `email`, `employeeNumber`, `jobTitle`, `hireDate`, `terminationDate`, `employmentStatus` | Master | Belongs To → Branch, Floor, Workspace, Department, Team |
| **Task** | `tasks` | `title`, `description`, `taskStatus`, `priority`, `dueDate`, `completedDate`, `estimatedHours`, `actualHours` | Transactional | Belongs To → Project; Assigned To → Employee |
| **Attendance Record** | `attendance_records` | `date`, `checkInTime`, `checkOutTime`, `attendanceStatus`, `hoursWorked`, `notes` | Transactional | Belongs To → Employee |
| **Satisfaction Survey** | `satisfaction_surveys` | `date`, `score` (1-100), `feedback` | Transactional | Belongs To → Branch |
| **Branch Financial** | `branch_financials` | `date`, `revenue`, `expenses` | Transactional | Belongs To → Branch |

---

## 2. Output Entities (Calculated/Stored Data)

These are the KPI snapshot tables generated by the Analytics service.

| Entity Name | Collection Name | Key Attributes | Purpose |
|-------------|-----------------|----------------|---------|
| **Employee KPI** | `employee_kpis` | `employee` (FK), `date`, `tasksCompleted`, `tasksTotal`, `attendanceRate`, `performanceScore`, `productivityScore`, `projectsCompleted` | Daily individual performance snapshot |
| **Floor KPI** | `floor_kpis` | `floor` (FK), `date`, `occupancyRate`, `productivityScore` | Daily floor utilization and average productivity |
| **Branch KPI** | `branch_kpis` | `branch` (FK), `date`, `revenue`, `revenueTarget`, `productivityScore`, `satisfactionScore`, `growthRate`, `performanceRating`, `joinedCount`, `leftCount`, `employeeCount`, `overdueTaskCount` | Daily branch-wide performance snapshot |

---

## 3. Detailed Schema Definitions

### 3.1 Employee KPI (`employee_kpis`)

| Attribute | Type | Constraints | Description |
|-----------|------|-------------|-------------|
| `id` | Integer | PK, Auto | Primary key |
| `employee` | Relation | FK → employees | Reference to the employee |
| `date` | Date | Required | KPI calculation date |
| `tasksCompleted` | Integer | Default: 0 | Count of completed tasks |
| `tasksTotal` | Integer | Default: 0 | Total tasks assigned |
| `attendanceRate` | Decimal | 0-100 | Percentage of days marked present |
| `performanceScore` | Decimal | 0-100 | Weighted score: (taskRate × 0.5) + (attendanceRate × 0.5) |
| `productivityScore` | Decimal | 0-100 | Weighted score: (taskRate × 0.7) + (attendanceRate × 0.3) |
| `projectsCompleted` | Integer | Default: 0 | Count of completed projects |

### 3.2 Floor KPI (`floor_kpis`)

| Attribute | Type | Constraints | Description |
|-----------|------|-------------|-------------|
| `id` | Integer | PK, Auto | Primary key |
| `floor` | Relation | FK → floors | Reference to the floor |
| `date` | Date | Required | KPI calculation date |
| `occupancyRate` | Decimal | 0-100 | (Employee count / Total workspace capacity) × 100 |
| `productivityScore` | Decimal | 0-100 | Average of all employee productivity scores on this floor |

### 3.3 Branch KPI (`branch_kpis`)

| Attribute | Type | Constraints | Description |
|-----------|------|-------------|-------------|
| `id` | Integer | PK, Auto | Primary key |
| `branch` | Relation | FK → branches | Reference to the branch |
| `date` | Date | Required | KPI calculation date |
| `revenue` | Decimal | Default: 0 | Actual revenue from Branch Financials |
| `revenueTarget` | Decimal | Default: 0 | Target revenue from Branch master |
| `productivityScore` | Decimal | 0-100 | Average of all floor productivity scores |
| `satisfactionScore` | Decimal | 0-100 | Average of satisfaction survey scores |
| `growthRate` | Decimal | - | ((Today Productivity - Yesterday) / Yesterday) × 100 |
| `performanceRating` | Enum | excellent, good, average, poor | Based on weighted score thresholds |
| `joinedCount` | Integer | Default: 0 | Employees hired this month |
| `leftCount` | Integer | Default: 0 | Employees terminated this month |
| `employeeCount` | Integer | Default: 0 | Total active employees |
| `overdueTaskCount` | Integer | Default: 0 | Tasks past due date |
